## 2. الأطراف الفاعلة — Actors

### 2.1 الأطراف الموثوقون

| الطرف | الوصف | القدرة |
|-------|-------|--------|
| **المتبرع (Donor)** | مستخدم مسجل يبحث عنه للتبرع | قراءة/كتابة بياناته فقط |
| **المريض/الباحث (Patient/Seeker)** | مستخدم يبحث عن متبرع | بحث، طلب اتصال، تقييم |
| **المطور (Developer)** | فريق التطوير | وصول تقني (محدود عبر RLS) |
| **DPO** | مسؤول حماية البيانات | إشراف على الامتثال |

### 2.2 الأطراف غير الموثوقون

| الطرف | الوصف | القدرة |
|-------|-------|--------|
| **المتسلل (Attacker)** | جهة خارجية تحاول الوصول | لا صلاحيات افتراضية |
| **المستخدم الخبيث (Malicious User)** | مستخدم مسجل ي滥用 النظام | صلاحيات مستخدم عادي |
| **المتصفح (Browser)** | الوسيط بين المستخدم والخادم | لا صلاحيات مباشرة |
| **Supabase** | مزود البنية التحتية | وصول إداري (موثق) |

## 3. حدود الثقة — Trust Boundaries

### 3.1 حدود الثقة

```
┌─────────────────────────────────────────────────────────────────┐
│                        حدود الثقة الخارجية                       │
│  ┌───────────────┐                                              │
│  │   المستخدم    │ ─── حدود الثقة 1 ───▶                       │
│  │  (Browser)    │     (HTTPS)                                  │
│  └───────────────┘                                              │
│         │                                                       │
│         ▼                                                       │
│  ┌───────────────┐                                              │
│  │     CDN       │ ─── حدود الثقة 2 ───▶                       │
│  │  (Supabase)   │     (API Gateway)                            │
│  └───────────────┘                                              │
│         │                                                       │
│         ▼                                                       │
│  ┌───────────────┐                                              │
│  │   Supabase    │ ─── حدود الثقة 3 ───▶                       │
│  │  (Edge Fn)    │     (Internal Network)                       │
│  └───────────────┘                                              │
│         │                                                       │
│         ▼                                                       │
│  ┌───────────────┐                                              │
│  │  PostgreSQL   │                                              │
│  │  (Database)   │                                              │
│  └───────────────┘                                              │
│                                                                 │
│                    حدود الثقة الداخلية                          │
└─────────────────────────────────────────────────────────────────┘
```

### 3.2 نقاط الفحص عند كل حد

| الحد | نقطة الفحص | الآلية |
|------|-----------|--------|
| **1. المتصفح → CDN** | المصادقة، التشفير | HTTPS + JWT tokens |
| **2. CDN → Edge Functions** | التحقق من الصلاحيات | API Gateway + Rate Limiting |
| **3. Edge Functions → PostgreSQL** | RLS + مصادقة المستخدم | Row Level Security |
| **4. Supabase → النسخ الاحتياطية** | التشفير | AES-256 at rest |

## 5. تحليل STRIDE —威胁

### 5.1 التزوير (Spoofing)

|---|---------|---------------|---------------|----------------|-------------------|---------|
| S1 | انتحال هوية مستخدم | بيانات المتبرعين | 🔴 حرج | JWT tokens + Supabase Auth | منخفضة | ✅ كافٍ |
| S2 | انتحال هوية مسؤول | النظام بالكامل | 🔴 حرج | لا يوجد دور مسؤول مُعرَّف | متوسطة | تعريف أدوار واضحة |
| S3 | هجوم Man-in-the-Middle | حركة المرور | 🟠 مرتفع | HTTPS (TLS 1.2+) | منخفضة | ✅ كافٍ |
| S4 | تسجيل دخول ببيانات مسروقة | حساب المستخدم | 🟠 مرتفع | OTP + rate limiting | منخفضة | تنفيذ MFA |

### 5.6 امتياز غير مصرح به (Elevation of Privilege)

| # | التهديد | الأصل المُهدد | مستوى المخاطرة | التخفيف الحالي | المخاطرة المتبقية | التوصية |

| E1 | صعود صلاحيات عبر RLS bypass | قاعدة البيانات | 🔴 حرج | RLS policies + testing | منخفضة | اختبار RLS بشكل دوري |
